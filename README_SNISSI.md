# SNISSI - Sistema Operativo Cognitivo Multiagente para Estudios Profesionales\n\n## Descripción General\n\nSNISSI es una plataforma de gestión de payroll asistida por agentes de inteligencia artificial que automatiza y supervisa el ciclo completo de liquidación de nómina con trazabilidad total y control humano sobre decisiones críticas.\n\n## Características Principales\n\n### 1. **Agentes IA Multiagente**\n\nCuatro agentes especializados operan en coordinación:\n\n- **Agente Interpretador Normativo**: Analiza cambios normativos y detecta impacto\n- **Agente Preliquidador**: Calcula conceptos y genera borradores\n- **Agente Auditor**: Detecta anomalías y valida coherencia\n- **Agente Comunicador**: Procesa novedades y genera alertas\n\n### 2. **Panel de Supervisión Humana (Human-in-the-Loop)**\n\n- Aprobación, rechazo y modificación de resultados de agentes\n- Conversión automática de decisiones en aprendizaje organizacional\n- Trazabilidad completa de todas las decisiones\n\n### 3. **ADN Organizacional**\n\nRepositorio centralizado de:\n- Políticas y directrices\n- Reglas de cálculo\n- Criterios de validación\n- Plantillas de documentos\n- Aprendizajes acumulados\n- Mejores prácticas\n\n### 4. **Trazabilidad Completa**\n\n- Log de auditoría por caso\n- Registro de cada acción, agente y decisión\n- Timeline visual de etapas\n- Exportación de reportes\n\n### 5. **Flujo de Trabajo Estructurado**\n\n```\nCOLLECTION → ANALYSIS → PRE_LIQUIDATION → AUDIT → APPROVAL → EMISSION → PRESENTATION → CLOSED\n```\n\n### 6. **Gestión de Documentos**\n\n- Upload seguro a S3\n- Referencias persistentes en BD\n- Validación de tipos de archivo\n- Descarga desde UI\n\n### 7. **Notificaciones Automáticas**\n\n- Anomalías detectadas\n- Aprobación requerida\n- Etapas completadas\n- Errores de agentes\n\n## Arquitectura Técnica\n\n### Stack Tecnológico\n\n- **Frontend**: React 19 + Tailwind CSS 4 + TypeScript\n- **Backend**: Express 4 + tRPC 11 + Node.js\n- **Base de Datos**: MySQL/TiDB\n- **LLM**: Claude 3.5 Sonnet\n- **Autenticación**: Manus OAuth\n- **Almacenamiento**: S3 (Manus Storage)\n\n### Estructura de Base de Datos\n\n```\nusers\npayroll_cases\ncase_stages\nnews_items\ndna_organizational\nagent_executions\naudit_log\nhuman_approvals\ndocuments\nnotifications\n```\n\n### Routers tRPC\n\n- `payroll.cases`: Gestión de casos\n- `payroll.news`: Bandeja de novedades\n- `payroll.agents`: Ejecuciones de agentes\n- `payroll.dna`: ADN organizacional\n- `payroll.approvals`: Supervisión humana\n- `payroll.documents`: Gestión de documentos\n- `payroll.audit`: Trazabilidad\n- `payroll.notifications`: Notificaciones\n\n## Guías de Uso\n\n### Para Supervisores\n\nVer: `SUPERVISOR_GUIDE.md`\n\n- Revisar resultados de agentes\n- Tomar decisiones (aprobar/rechazar/modificar)\n- Registrar aprendizaje organizacional\n- Gestionar casos complejos\n\n### Para Administradores\n\nVer: `ADMIN_GUIDE.md`\n\n- Gestión del ADN organizacional\n- Configuración del sistema\n- Mantenimiento de base de datos\n- Resolución de problemas\n\n### Para Desarrolladores\n\nVer: `AGENTS_API.md` y `DNA_STRUCTURE.md`\n\n- API de agentes\n- Estructura del ADN\n- Integración con LLM\n- Extensión del sistema\n\n## Instalación y Configuración\n\n### Requisitos\n\n- Node.js 22.13.0+\n- pnpm 10.4.1+\n- MySQL 8.0+ o TiDB\n- Acceso a Claude API\n\n### Instalación\n\n```bash\n# Clonar repositorio\ngit clone <repo-url>\ncd sistema-multiagente-inteligente\n\n# Instalar dependencias\npnpm install\n\n# Configurar variables de entorno\ncp .env.example .env\n# Editar .env con credenciales\n\n# Ejecutar migraciones\npnpm drizzle-kit generate\npnpm drizzle-kit migrate\n\n# Iniciar servidor de desarrollo\npnpm dev\n```\n\n### Variables de Entorno\n\n```\nDATABASE_URL=mysql://user:password@localhost:3306/snissi\nJWT_SECRET=your-secret-key\nVITE_APP_ID=your-app-id\nOAUTH_SERVER_URL=https://api.manus.im\nBUILT_IN_FORGE_API_URL=https://api.manus.im\nBUILT_IN_FORGE_API_KEY=your-api-key\n```\n\n## Uso del Sistema\n\n### Flujo Principal\n\n1. **Crear Caso**: Ir a Gestión de Casos → Nueva Caso\n2. **Cargar Novedades**: Ir a Bandeja de Novedades → Cargar\n3. **Ejecutar Agentes**: Sistema ejecuta automáticamente según etapa\n4. **Revisar Resultados**: Ir a Supervisión Humana\n5. **Tomar Decisión**: Aprobar, rechazar o modificar\n6. **Registrar Aprendizaje**: Crear regla si es necesario\n7. **Avanzar Etapa**: Caso avanza automáticamente\n8. **Cerrar Caso**: Cuando todas las etapas están completas\n\n### Gestión del ADN\n\n1. Ir a ADN Organizacional\n2. Revisar reglas existentes\n3. Agregar nuevas reglas según necesidad\n4. Actualizar cuando hay cambios normativos\n5. Consolidar aprendizajes periódicamente\n\n## Pruebas\n\n### Ejecutar Tests\n\n```bash\npnpm test\n```\n\n### Tests Disponibles\n\n- `server/auth.logout.test.ts`: Tests de autenticación\n-
